СОСТАВ ПОСТАВКИ
----------------
1. RetailPro/RetailPro.pbip          — дашборд Power BI (3 роли)
2. powerbi_data/                   — данные для отчёта

10. dashboard.py + start_dashboard.bat — веб-дашборд (опционально)
11. data/sales.csv                    — пример структуры данных (заменить на свой)

ПАКЕТ ПРИЁМКИ (acceptance_package/)
-----------------------------------
Документы с доказательствами корректности и полноты дашборда:

  01_ПОДГОТОВКА_ДАННЫХ.txt      — шаги очистки и производные поля
  02_ПРОВЕРКА_KPI.txt           — сверка KPI с ручным расчётом (pandas)
  03_ФИЛЬТРЫ_И_ГРАФИКИ.txt      — перечень фильтров и графиков по ролям
  04_УПРАВЛЕНЧЕСКОЕ_РЕЗЮМЕ.txt  — 2–3 управленческих наблюдения
  ПАКЕТ_ПРИЕМКИ.txt             — сводный документ
  acceptance_summary.json       — машиночитаемая сводка KPI и выводов

Пересборка пакета:
  python prepare_retailpro_data.py --input sales.csv
  (или) python generate_acceptance_report.py --input sales.csv

ТРЕБОВАНИЯ К ИСХОДНЫМ ДАННЫМ
-----------------------------
Файл CSV или Excel с колонками (имена строго такие):
  date, region, category, manager, revenue, quantity,
  discount_amount, status, returns

Соответствие фильтров дашборда:
  Период           → date
  Источник/канал   → region (отдельной колонки source нет)
  Продукт/услуга   → category
  Менеджер         → manager
  Регион/сегмент   → region

KPI И ФОРМУЛЫ
-------------
  Выручка      = SUM(revenue)
  Заявки       = COUNT(строк) — каждая строка = обращение
  Продажи      = revenue>0, returns=0, status не отмена/возврат
  Конверсия %  = Продажи / Заявки
  Средний чек  = SUM(revenue продаж) / Продажи
  MoM          = сравнение последнего и предыдущего календарного месяца

Быстрый старт (Power BI)
------------------------
1. Установить Power BI Desktop (бесплатно).
2. Положить свой файл продаж как sales.csv в папку с проектом.
3. Дважды кликнуть prepare_powerbi.bat
4. Дважды кликнуть open_powerbi.bat
   (или открыть RetailPro\RetailPro.pbip)
5. В Power BI нажать «Обновить».
6. Перейти на вкладки:
   — Менеджер
   — Руководитель
   — Коммерческий директор

Роли и доступ к данным
----------------------
Менеджер              — видит только свои продажи
Руководитель          — видит продажи своего региона
Коммерческий директор — видит всю компанию

Файл powerbi_data/UserAccess.csv — таблица доступа.
Замените user_login на реальные email сотрудников компании.

Отдельные выгрузки по ролям:
  powerbi_data/by_role/director/
  powerbi_data/by_role/supervisor/<регион>/
  powerbi_data/by_role/manager/<менеджер>/

После публикации в Power BI Service:
  Моделирование → Просмотр как → выбрать роль для проверки.

Обновление данных (ежемесячно)
--------------------------------
1. Положить новый sales.csv
2. Запустить: python prepare_retailpro_data.py --input sales.csv
3. Запустить: python build_powerbi_project.py
4. Открыть RetailPro.pbip → Обновить

Проверка перед приёмкой
-----------------------
python test_all.py
Ожидается: «Все проверки пройдены»

Критерии приёмки MVP
--------------------
[ ] Данные ограничены последними 3 месяцами
[ ] KPI: выручка, заявки, продажи, конверсия, средний чек (+ возвраты) — с MoM
[ ] Фильтры: период, регион/источник, категория, менеджер
[ ] Графики: динамика по неделям, сравнение регионов, менеджеры, топ-5 продуктов
[ ] Таблица детализации + экспорт в Excel
[ ] Блок «Выводы» — 3 управленческих наблюдения
[ ] 3 роли: менеджер / руководитель / директор
[ ] Пакет acceptance_package/ с доказательствами сверки KPI
