================================================================================
RetailPro — инструкция для заказчика
================================================================================

>>> НАЧНИТЕ С ФАЙЛА: НАЧНИТЕ_ЗДЕСЬ.txt или ОПИСАНИЕ_РЕЗУЛЬТАТА.txt <<<

В корне архива находится описание результата работы с:
  • подтверждением корректности всех 5 KPI (выручка, заявки, продажи,
    конверсия, средний чек) и их отображением на дашборде;
  • подтверждением работы фильтров (период, источник, продукт, менеджер, регион);
  • управленческим резюме — 2–3 развёрнутых наблюдения.

СОСТАВ ПОСТАВКИ
----------------
1. НАЧНИТЕ_ЗДЕСЬ.txt               — точка входа для приёмки
2. ОПИСАНИЕ_РЕЗУЛЬТАТА.txt         — описание результата для заказчика
3. RetailPro/RetailPro.pbip          — дашборд Power BI (3 роли)
4. powerbi_data/                   — данные для отчёта
5. acceptance_package/             — детальные документы приёмки
6. prepare_retailpro_data.py         — подготовка данных из CSV/Excel
7. generate_acceptance_report.py     — пересборка пакета приёмки
8. build_powerbi_project.py          — пересборка отчёта (при смене данных)
9. dax_measures.txt                  — DAX-меры (справочник)
10. power_bi_visuals.txt              — настройка и проверка Power BI
11. role_access.py                    — логика ролей
12. dashboard.py + start_dashboard.bat — веб-дашборд (опционально)
13. data/sales.csv                    — пример структуры данных (заменить на свой)

ПАКЕТ ПРИЁМКИ (acceptance_package/)
-----------------------------------
  01_ПОДГОТОВКА_ДАННЫХ.txt      — шаги очистки и производные поля
  02_ПРОВЕРКА_KPI.txt           — сверка KPI с ручным расчётом (pandas)
  03_ФИЛЬТРЫ_И_ГРАФИКИ.txt      — перечень фильтров и графиков по ролям
  04_УПРАВЛЕНЧЕСКОЕ_РЕЗЮМЕ.txt  — развёрнутые управленческие наблюдения
  05_ПРОВЕРКА_ФИЛЬТРОВ.txt      — автотест работы фильтров
  ПАКЕТ_ПРИЕМКИ.txt             — сводный документ

ТРЕБОВАНИЯ К ИСХОДНЫМ ДАННЫМ
-----------------------------
Файл CSV или Excel с колонками (имена строго такие):
  date, region, category, manager, revenue, quantity,
  discount_amount, status, returns

Соответствие фильтров дашборда:
  Период           → date
  Источник/канал   → region (отдельной колонки source нет)
  Продукт/услуга   → category
  Менеджер         → manager
  Регион/сегмент   → region

KPI И ФОРМУЛЫ (подтверждены сверкой)
------------------------------------
  Выручка      = SUM(revenue)           — карточка на дашборде
  Заявки       = COUNT(строк)           — карточка на дашборде
  Продажи      = is_sale = 1            — карточка на дашборде
  Конверсия %  = Продажи / Заявки       — карточка на дашборде
  Средний чек  = Выручка продаж / Продажи — карточка на дашборде

Быстрый старт (Power BI)
------------------------
1. Установить Power BI Desktop (бесплатно).
2. Положить свой файл продаж как sales.csv в папку с проектом.
3. Дважды кликнуть prepare_powerbi.bat
4. Дважды кликнуть open_powerbi.bat
   (или открыть RetailPro\RetailPro.pbip)
5. В Power BI нажать «Обновить».
6. Перейти на вкладки:
   — Менеджер
   — Руководитель
   — Коммерческий директор
7. Блок «Управленческое резюме» — в нижней части каждой страницы.

Роли и доступ к данным
----------------------
Менеджер              — видит только свои продажи
Руководитель          — видит продажи своего региона
Коммерческий директор — видит всю компанию

Обновление данных (ежемесячно)
--------------------------------
1. Положить новый sales.csv
2. Запустить: python prepare_retailpro_data.py --input sales.csv
3. Запустить: python build_powerbi_project.py
4. Открыть RetailPro.pbip → Обновить

Проверка перед приёмкой
-----------------------
python test_all.py
Ожидается: «Все проверки пройдены»

Критерии приёмки MVP
--------------------
[ ] ОПИСАНИЕ_РЕЗУЛЬТАТА.txt с подтверждением KPI и фильтров
[ ] Управленческое резюме: 2–3 развёрнутых наблюдения
[ ] KPI на дашборде: выручка, заявки, продажи, конверсия, средний чек
[ ] Фильтры: период, регион/источник, категория, менеджер
[ ] Графики: динамика, регионы, менеджеры, топ-5 продуктов
[ ] 3 роли: менеджер / руководитель / директор
